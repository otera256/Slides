# 時系列データと言語モデル

## 時系列データ
- 時系列データは、時間の経過に伴うデータの変化
- 前回扱った画像データが2次元などの空間的な構造を持つのに対し、時系列データは時間軸に沿った構造を持つ
  - 例: 株価の変動、気温の変化、音声、テキストデータなど
### 時系列データの特徴
- 時間的な依存関係が存在する
- ノイズや外れ値が含まれることが多い
- 季節性やトレンドなどのパターンが存在することがある
- 非定常性（時間とともに統計的な性質が変化すること）がある
- 多くの時系列データは、過去の値に基づいて未来の値を予測することが目的となる
### 基本的な戦略
- 過去の値を特徴量として使用する
  - 例: 株価の予測では、過去の株価や取引量を特徴量として使用する
- iステップ前ごとにモデルを用意する？
  - 例: 1ステップ前の値を予測するモデル、2ステップ前の値を予測するモデルなど
- 非常に多くのモデルが必要になる可能性があるため、効率的な方法を考える必要がある
- 前回扱った畳み込みニューラルネットワークの１次元版のように**時間方向**のパラメータ共有を行うのが基本
### RNN（リカレントニューラルネットワーク）
- 時系列データの処理に特化したニューラルネットワークの一種
まず最も単純なRNNの構造を定義し、そのモデルの学習を行う上での問題点を説明する。
$\sigma$を活性化関数、$W_{h}$を隠れ状態の重み、$W_{i}$を入力の重み、$W_{o}$を出力の重み、$h_t$を時刻$t$の隠れ状態、$x_t$を時刻$t$の入力、$y_t$を時刻$t$の出力とする。
$$h_t = \sigma(W_{h} h_{t-1} + W_{i} x_t)$$
$$y_t = W_{o} h_t$$
- RNNは、前の隠れ状態$h_{t-1}$と現在の入力$x_t$を組み合わせて新しい隠れ状態$h_t$を生成し、その隠れ状態から出力$y_t$を生成する
- つまり再帰的（Recurrent）な構造を持ち、時間的な依存関係をモデル化することができる

### RNNの学習方法
- RNNの学習は、通常のニューラルネットワークと同様に、誤差逆伝播法を使用して行う
- 計算グラフを逆順に辿って勾配を計算するという点は同じだが、計算を効率化するためにMLPなどRNN以外の層には時間方向をバッチと同様にまとめて一気に計算してしまう。その都合上、RNNだけテンソルを時間方向に展開して計算する必要がある。
- 誤差逆伝播法を時間方向に適用する必要がある（Backpropagation Through Time, BPTT）
- BPTTでは、RNNの隠れ状態が時間的に依存しているため、時間方向に沿って勾配を計算する必要がある
- 地味に実装が面倒で、特に直列な処理となるためGPUの恩恵を受けにくくメモリの使用量も多くなりがちであることに注意する必要がある
### 勾配消失と勾配爆発
- 第4回で説明したようにニューラルネットワークが安定的に学習するためにはモデル内部のアクティベーションや勾配の分散が適切な範囲に収まる必要がある
- 簡単のために活性化関数が恒等写像である場合について考えると、RNNの隠れ状態の更新は以下のようになる。
$$h_t = W_{h} h_{t-1} + W_{i} x_t$$
- $W_{h}$の固有値が1より大きい場合、その固有値に属する成分のアクティベーションは時間とともに指数関数的に増加し、勾配は時間とともに爆発する（勾配爆発）
- $W_{h}$の固有値が1より小さい場合、その固有値に属する成分のアクティベーションは時間とともに指数関数的に減少し、勾配は時間とともに消失する（勾配消失）
- 非線形関数を使用する場合も同様の問題が発生する可能性がある
- 勾配消失や勾配爆発は、RNNの学習を非常に困難にするため、これらの問題を解決するための手法がいくつか提案されている
- アーキテクチャの工夫
  - LSTM（Long Short-Term Memory）やGRU（Gated Recurrent Unit）などの特殊なRNNアーキテクチャは、勾配消失や勾配爆発の問題を緩和するためのゲート機構を導入している
- 勾配クリッピング
  - 勾配の大きさが一定の閾値を超えないようにすることで、勾配爆発を防止する手法
  - 例えば、勾配のノルムが閾値を超える場合に、勾配をスケーリングしてノルムを閾値に合わせる方法がある
  - ある意味正規化の一種とも言える
- 注意深い重みの初期化
  - RNNの重みを適切に初期化することで、勾配消失や勾配爆発の問題を緩和することができる
  - 例えば活性化関数が恒等写像の場合は、$W_{h}$の固有値が1に近くなるように初期化することが望ましい
    - 補足: 固有値がすべて1となる正方行列は特殊直行行列である。
    - 補足: 活性化関数が非線形の場合はかなり複雑な議論となってしまい、RNNではこの種の議論が一般的ではないが、状態空間モデル（State Space Model）などの分野ではこの種の議論が一般的である。
線形代数の復習になるので少し面白い例を紹介する。$W_{h}$がエルミート行列である場合は$W_{h} = U^{T} D U$とユニタリ行列$U$と対角行列$D$を用いて対角化できるため、RNNの隠れ状態の更新は以下のようになる。
$$h_t = W_{h} h_{t-1} + W_{i} x_t$$
$$h_t = U^{T} D U h_{t-1} + W_{i} x_t$$
$$h_t = h_0 + \sum_{k=1}^{t} U^{T} D^{t-k} U W_{i} x_k$$
$D^{t-k}$は計算が容易であり、またこの種の畳み込み計算はFFTを用いて高速に計算できるためこの種の理論を活用したモデルも研究されている。（例: S4モデル）
### LSTMとGRU
- LSTM（Long Short-Term Memory）とGRU（Gated Recurrent Unit）は、RNNの特殊なアーキテクチャであり、勾配消失や勾配爆発の問題を緩和するためのゲート機構を導入している
- LSTMは、入力ゲート、忘却ゲート、出力ゲートの3つのゲートを持ち、隠れ状態とセル状態を分けて管理することで、長期的な依存関係を学習することができる
$h_t$を隠れ状態、$c_t$をセル状態、$i_t$を入力ゲート、$f_t$を忘却ゲート、$o_t$を出力ゲートとすると、LSTMの更新は以下のようになる。
$$i_t = \sigma(W_{i} x_t + U_{i} h_{t-1} + b_i)$$
$$f_t = \sigma(W_{f} x_t + U_{f} h_{t-1} + b_f)$$
$$o_t = \sigma(W_{o} x_t + U_{o} h_{t-1} + b_o)$$
$$\tilde{c}_t = \tanh(W_{c} x_t + U_{c} h_{t-1} + b_c)$$
$$c_t = f_t \odot c_{t-1} + i_t \odot \tilde{c}_t$$
$$h_t = o_t \odot \tanh(c_t)$$
それぞれのゲートの役割は以下の通りである。
- 入力ゲート($i_t$): 現在の入力$x_t$と前の隠れ状態$h_{t-1}$に基づいて、セル状態にどの程度新しい情報を追加するかを決定する
- 忘却ゲート($f_t$): 現在の入力$x_t$と前の隠れ状態$h_{t-1}$に基づいて、セル状態からどの程度古い情報を忘れるかを決定する
- 出力ゲート($o_t$): 現在の入力$x_t$と前の隠れ状態$h_{t-1}$に基づいて、隠れ状態からどの程度の情報を出力するかを決定する
セル状態の更新に注目すると、忘却ゲート$f_t$がセル状態$c_{t-1}$をスケーリングし、入力ゲート$i_t$が新しい候補セル状態$\tilde{c}_t$をスケーリングしていることがわかる。これにより、（忘却ゲートの値が適切であるという条件下で）指数関数的な減衰や増加を防止することができるため、長期的な依存関係を学習することができる。
- GRUは、更新ゲートとリセットゲートの2つのゲートを持ち、隠れ状態を直接管理することで、LSTMよりもシンプルな構造を持つ
$h_t$を隠れ状態、$z_t$を更新ゲート、$r_t$をリセットゲートとすると、GRUの更新は以下のようになる。
$$z_t = \sigma(W_{z} x_t + U_{z} h_{t-1} + b_z)$$
$$r_t = \sigma(W_{r} x_t + U_{r} h_{t-1} + b_r)$$
$$\tilde{h}_t = \tanh(W_{h} x_t + U_{h} (r_t \odot h_{t-1}) + b_h)$$
$$h_t = (1 - z_t) \odot h_{t-1} + z_t \odot \tilde{h}_t$$
それぞれのゲートの役割は以下の通りである。
- 更新ゲート($z_t$): 現在の入力$x_t$と前の隠れ状態$h_{t-1}$に基づいて、隠れ状態をどの程度更新するかを決定する
- リセットゲート($r_t$): 現在の入力$x_t$と前の隠れ状態$h_{t-1}$に基づいて、隠れ状態をどの程度リセットするかを決定する
GRUもLSTMと同様に、更新ゲート$z_t$が隠れ状態$h_{t-1}$をスケーリングし、リセットゲート$r_t$が新しい候補隠れ状態$\tilde{h}_t$の計算に影響を与えることで、長期的な依存関係を学習することができる

## 言語モデル
- 言語モデルは、自然言語のテキストデータを処理するためのモデルであり対話型AIの基盤技術の一つ
- 言語をモデル化していればそれは言語モデルなのだが、一般的には、ある単語の列が与えられたときに次の単語を予測するタスクを指すことが多い
  - 例えば、"The apple is"という単語の列が与えられたときに、次の単語を予測するタスクでは、"red"や"ripe"などの単語が予測されることが望ましい
  - 一応拡散言語モデルのような文章全体の"自然さ"を評価するタスクもあるが、基本的には次の単語を予測するタスクが言語モデルの基本的なタスクである
- 言語モデルは、自然言語処理の様々なタスク（例: 機械翻訳、要約、質問応答など）において重要な役割を果たす
- 別に機械学習でなくとも、言語モデルは古くから研究されてきた分野であり、機械学習を用いた言語モデルもその一部である
  - n-gramモデル: 単語の列をn個の単語のグループ（n-gram）に分割し、次の単語を予測するためにn-gramの出現頻度を利用する
- 機械学習においては以下のような流れで言語モデリングをするのが最も典型的である
  1. テキストデータを収集する
  2. テキストデータを1文字ごとや1単語ごとなどのトークンに分割する（Tokenization）
  3. トークンを数値に変換する（例: ワンホットエンコーディング、埋め込みベクトルなど）
  4. 数値化されたトークンをモデルに入力し、次のトークンを予測するタスクを定義する（自己教師あり学習）
  5. モデルを学習する
- 言語モデルの性能は、予測されたトークンの確率分布と実際のトークンの確率分布とのクロスエントロピー損失を最小化することで評価されることが多い
- 言語モデルの性能を評価するための指標として、Perplexityがよく使用される
  - Perplexityは、モデルが次のトークンをどれだけ予測できているかを示す指標であり、値が小さいほど性能が良いことを示す
  - Perplexityは、クロスエントロピー損失の指数関数で定義されるため、クロスエントロピー損失が小さいほどPerplexityも小さくなる

### TokenizationとEmbedding
- Tokenizationは、テキストデータをトークンに分割するプロセスであり、言語モデルの前処理の一部である
- トークンの単位は、文字、単語、サブワードなど様々な単位がある
- Tokenの数はそのモデルが直接扱うことのできる語彙のサイズであるため、モデルの性能や計算コストに大きな影響を与える
- かつてはword2vecのようなモデルと独立して学習された単語埋め込みが一般的であったが、現在ではTransformerのようなモデルと一体で学習されることが多い
- 埋め込みベクトルは、トークンを数値化するための方法であり、トークンを高次元の連続空間にマッピングする
- 埋め込みベクトルは、トークンの意味的な類似性を捉えることができるため、言語モデルの性能を向上させることができる
- 埋め込みベクトルの次元数は、モデルの性能や計算コストに大きな影響を与えるため、適切な次元数の選択が重要である
- 埋め込みベクトルの学習は、通常のニューラルネットワークの学習と同様に、誤差逆伝播法を使用して行う

### Transformerの紹介
- Transformerは、2017年に"Attention is All You Need"という論文で提案されたニューラルネットワークのアーキテクチャであり、言語モデルの分野で革命をもたらした
- ChatGPTなどが生まれたのもTransformerの登場が大きなきっかけである
- Transformerは、自己注意機構（Self-Attention）を利用して、入力シーケンス内のすべてのトークンが互いにどの程度関連しているかを計算することで、長期的な依存関係を効果的にモデル化することができる
Transformerの核となるAttention機構は以下のように定義できる。$Q$をクエリ、$K$をキー、$V$をバリューとすると、Attentionは以下のように計算される。
$$\text{Attention}(Q, K, V) = \text{softmax}\left(\frac{QK^T}{\sqrt{d_k}}\right) V$$
ここで、$d_k$はキーの次元数であり、スケーリングファクターとして使用される。Attentionは、クエリとキーの内積を計算し、その結果をスケーリングしてからソフトマックス関数を適用することで、各トークンが他のトークンにどの程度注意を払うべきかを示す重みを計算する。そして、その重みをバリューに乗じて出力を生成する。シーケンス長Nの入力に対して、Attentionの計算はO(N^2)の時間複雑度を持つため、長いシーケンスに対しては計算コストが高くなることがあるが、それよりもGPUの並列計算の恩恵を受けやすいことが大きな利点である。
これも説明だけだとわかりにくいので[こちらの動画](https://youtu.be/j3_VgCt18fA?si=V50Pjw_xxfAkC-BR)を見てTransformerの全体像を掴む
- Transformerは、自然言語処理の様々なタスク（例: 機械翻訳、要約、質問応答など）において最先端の性能を達成している
- 最新のAIモデルのデフォルトスタンダードになっているだけあって細かいモデルのカスタマイズ要素が無数にあるが、Attention機構そのものは非常にシンプルであるため、Transformerの基本的な構造を理解することは、言語モデルの理解にとって重要である
- 機械学習関連の論文を読み始めるのであればまず[Attention is All You Need](https://arxiv.org/abs/1706.03762)を読むことを強くおすすめする
